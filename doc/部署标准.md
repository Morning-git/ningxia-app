# 前端项目部署标准（GitHub + Vercel）

适用于 Vue/Vite 等前端项目的线上演示部署，产出固定可访问链接，无需本地启动。

---

## 一、流程总览

1. 在 GitHub 创建空仓库（不勾选 README/.gitignore）
2. 本地项目绑定远程并推送（推荐 SSH）
3. 在 Vercel 导入该仓库并部署
4. 使用 Vercel 提供的固定链接作为演示地址

---

## 二、GitHub 准备

### 2.1 创建仓库

- **Repository name**：与项目名一致（如 `ningxia-app`），仅字母、数字、`-`、`_`
- **Visibility**：Public 或 Private 按需选择
- **不要勾选**：Add a README、.gitignore、license（避免与本地冲突）

### 2.2 本地绑定与推送（HTTPS 或 SSH 二选一）

**推荐：SSH（一次配置，长期免输密码）**

```bash
cd /path/to/your-project

# 若尚未初始化
git init

# 绑定远程（SSH）
git remote add origin git@github.com:<你的用户名>/<仓库名>.git
# 若 origin 已存在：git remote set-url origin git@github.com:...

git add .
git commit -m "init project"
git branch -M master
git push -u origin master
```

**SSH 首次使用需配置：**

```bash
ssh-keygen -t ed25519 -C "你的邮箱或用户名"
cat ~/.ssh/id_ed25519.pub
```

将输出的整行公钥添加到 GitHub：**Settings → SSH and GPG keys → New SSH key**。

## 三、Vercel 部署

### 3.1 登录与授权

- 打开 https://vercel.com ，使用 **GitHub 账号登录**（便于直接选仓库）
- 若用 Google 登录，需在 Vercel 内再 **Connect GitHub** 才能导入 GitHub 仓库

### 3.2 导入项目

- **Add New → Project**
- 在列表中找到目标仓库，点击 **Import**
- 若提示 “Install the GitHub application”，点击 **Install**，在 GitHub 授权页面选择账号与仓库权限（可只选当前仓库）

### 3.3 构建配置（Vite 项目示例）

| 配置项 | 值 |
|--------|-----|
| Framework Preset | Vite |
| Root Directory | `./`（仓库根目录即项目根目录） |
| Build Command | `npm run build` |
| Output Directory | `dist` |

确认后点击 **Deploy**。部署完成后会得到固定链接，如：`https://<项目名>.vercel.app`。

### 3.4 本地修改后的更新流程（标准操作）

> 适用于：已经部署过一次，后续只是在本地改代码，需要把最新代码同步到线上演示环境。

1. **在本地修改代码并自测**
   ```bash
   # 进入项目目录
   cd /path/to/your-project

   # 可选：本地预览
   npm install        # 首次或依赖变更时
   npm run dev
   ```
2. **提交到 Git（仅提交有意义的改动）**
   ```bash
   git status                    # 查看变更
   git add .                     # 或按文件粒度选择 add
   git commit -m "描述本次改动"
   ```
3. **推送到远程 master 分支**
   ```bash
   git push origin master
   ```
4. **等待 Vercel 自动构建并发布**
   - 推送完成后，Vercel 会自动触发一次新的部署；
   - 无需手动点 Deploy，几分钟内完成；
   - 部署成功后，**之前的固定链接不变**，内容自动更新为最新代码。

> 如需回滚到旧版本，可在 Vercel 项目 Deployments 页面选择某个历史部署，点击 **Promote to Production**。

## 四、注意点与常见错误（必读）

以下为实际部署中易出现的错误及对应处理方式，请按顺序排查。

### 4.1 GitHub 认证相关

| 现象 | 原因 | 处理方式 |
|------|------|----------|
| `Password authentication is not supported for Git operations` / `Invalid username or token` | GitHub 已不再支持用账号密码 push，必须用 Token 或 SSH | **HTTPS**：在 GitHub 生成 Personal Access Token (classic)，勾选 `repo`，push 时“密码”处粘贴 Token；**推荐改用 SSH**（见 2.2） |
| `Permission to xxx denied to xxx` / `error: 403` | Token 权限不足或未勾选 repo；或用了 fine-grained token 但未授权该仓库 | 重新生成 **Tokens (classic)**，勾选 **repo** 整项；或改用 SSH 推送 |
| `failed to authenticate via web browser: i/o timeout`（gh auth login） | 网络无法稳定访问 GitHub OAuth | 换网络（如手机热点）再试，或放弃 gh 登录，直接用 Token/SSH |

### 4.2 网络相关

| 现象 | 原因 | 处理方式 |
|------|------|----------|
| `Failed to connect to github.com port 443` / `Couldn't connect to server` | 当前网络无法访问 GitHub（防火墙/代理/地区限制） | 换网络（手机热点、家庭宽带）再 push；或改用国内 Git 托管（Gitee 等），再在 Vercel 中连接对应平台（若支持） |
| `Error in the HTTP2 framing layer` | 网络不稳定或代理干扰 HTTPS/HTTP2 | 同上一行，优先换网络重试 |

### 4.3 仓库内容与大小

| 现象 | 原因 | 处理方式 |
|------|------|----------|
| `File xxx is 149.06 MB; this exceeds GitHub's file size limit of 100.00 MB` / `GH001: Large files detected` | 单文件超过 GitHub 100MB 限制 | 从项目中删除或移出该大文件（如安装包、压缩包）；若历史里已有，可 `rm -rf .git && git init` 后重新 add/commit/push（舍弃旧历史） |
| `remote origin already exists` | 重复执行了 `git remote add origin` | 改用 `git remote set-url origin <新地址>` 更新地址 |

### 4.4 Vercel 相关

| 现象 | 原因 | 处理方式 |
|------|------|----------|
| 点击 “Continue with GitHub” 后一直停在 “Authenticating” / 白屏 | 网络无法访问 GitHub OAuth 或浏览器拦截 | 换网络、换浏览器或隐身模式；或先用 Email/Google 登录 Vercel，再在 **Settings → Git → Connect GitHub** 绑定 GitHub |
| 部署后页面 404 或空白 | 构建输出目录或框架识别错误 | 在 Vercel 项目 **Settings → General** 中确认 **Output Directory** 为 `dist`，**Framework Preset** 为 Vite（或对应框架） |

### 4.5 推送前自检清单

- [ ] 已删除或排除超过 100MB 的文件（含 `doc/`、安装包等）
- [ ] 远程地址正确：`git remote -v` 显示为 `git@github.com:...` 或 `https://github.com/...`
- [ ] 认证方式已就绪：SSH 已添加公钥到 GitHub，或 HTTPS 已准备好 Token
- [ ] 当前网络能访问 github.com（可浏览器打开 https://github.com 测试）

## 五、交付物

- **演示链接**：Vercel 提供的 Production 域名，如 `https://ningxia-app.vercel.app`
- **更新方式**：向 `master` 分支 push 即可自动更新该链接内容
